Objective: Introduce Firedancer, Multi-Local Consensus & Validators.

Fogo is built around ultra-fast technology and sometimes difficult to understand. This topic is a bit technical but necessary to understand how fogo works and why.


Firedancer Client :

Firedancer is a high-performance, independent validator designed to improve speed, scalability, and resilience.

  • Built in C by Jump Crypto.
  • Fully parallelized with low-level optimization.
  • Removes bottlenecks from traditional client implementations.

Multi-Local Consensus :

Multi-Local Consensus is a zone-based distributed consensus mechanism designed to enhance network performance and scalability.

  • Validators co-locate in strategic global zones.
  • Each "zone" runs local consensus for ultra-low latency trading during active hours.
  • "Follow-the-sun" model: consensus shifts dynamically to serve active markets.
  • Fallback global consensus ensures resilience.

Validator Structure:

Validator Structure is optimized for fast and secure consensus.

  • Initially permissioned, curated for high reliability.
  • Stake-weighted governance for validator rotation and zone creation.
  • Two-tier keys: global identity and zone-specific.
  • Initially 20-50 validators.
